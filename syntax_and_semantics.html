Syntax and Semantics

  • Syntax

    • Need for parenthesis around all forms. Decreases writability and gives greater chance of trivial mistakes. Would prefer parenthesis allowed in these same places if desired but not required.
    • Function definitions do not define argument type or return type
    • Code is generally very concise

  • Semantics

    • When creating local names using Let, they have Lexical Scope
    • Constants can be declared when using def to create a Var, in which case you will add metadata to the Var using const.
      This metadata can be added to different kinds of objects as well, and is stored as a map to the Var.
    • Garbage collection is done by taking advatage of the JVM.
